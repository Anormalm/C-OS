# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch and the most recent tagged release.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| `< latest release` | No |

## Reporting A Vulnerability

Please report vulnerabilities privately:

1. Open a private security advisory in GitHub Security Advisories for this repository.
2. Or email the maintainer with:
   - reproduction steps
   - impact assessment
   - suggested fix (if known)

Expected process:

1. Acknowledgement within 72 hours.
2. Initial triage within 7 days.
3. Patch target based on severity.
4. Coordinated disclosure after fix release.

Do not post exploit details in public issues before a patch is available.
